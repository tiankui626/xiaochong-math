## 流程一：新课件整理

用户说"第N讲上完了，帮我整理"时执行。

### Step 1：PDF转图片

使用 `scripts/pdf_to_images.py` 脚本，或手动执行：

```python
import fitz, os
out = f'附件/第{N}讲'
os.makedirs(out, exist_ok=True)
for pdf_path, prefix in pdfs:
    doc = fitz.open(pdf_path)
    for i, page in enumerate(doc):
        page.get_pixmap(dpi=150).save(f'{out}/第{N}讲_{prefix}_p{i+1}.png')
    doc.close()
```

> **命名约定**：图片文件名必须带讲次前缀（如 `第6讲_课后小测_题目_p1.png`），Obsidian 按文件名全局匹配，不区分目录。

### Step 2：读取图片识别内容

- 课后小测：读题目版图片获取完整题目和数字
- 课后小测解析版：读取答案和解析
- 加油站最后一页：答案汇总页，按"举一反三/基础过关/活学活用/勇攀高峰"分类

### Step 3：生成知识点笔记

文件：`知识点/第N讲 {主题}.md`

```yaml
---

lecture: N

```

内容结构：核心知识点（`> [!tip]` callout）、解题策略（Mermaid 流程图）、经典题型、易错点（`> [!warning]`）、相关链接。

### Step 4：生成课后小测 & 加油站

加油站**必须先读答案汇总页确认板块结构**，不同讲次可能缺少某些板块，必须逐页检查确认，不能只看最后一页。

## 流程四：互动出题（小朋友直接对话模式）

小朋友说"出一道题"或"我要做题"时执行，适用于 OpenClaw 等对话场景。

### 出题规则

**范围限定：**
- 只围绕第1-6讲内容出题：方田探秘、年年有余、何时无余、深思熟虑、几何计数进阶、平行四边形与梯形
- 不出超纲题、不出与讲次无关的题

**出题要求：**
1. **直接出题**，立即出一道，不需确认
2. **随机薄弱点**：从当前讲次错题本中随机选一个薄弱点，不按固定顺序
3. **一次只出一道**，等回答后再出下一道
4. **实时保存**：每道题写入 `题库/变式练习/` 对应文件
5. **不主动给提示**，等虫虫说没头绪时再给
6. **出一道问一道**，每题后问"要继续下一题吗"
7. **几何题优先用原图**：先从 `附件/第N讲/` 找，其次文字描述，最后才用 image_generate

### 答案检查规则

**核心要求：至少3步中间过程**

- **三步要求**：解答必须有完整中间过程，至少3步
  - ✅ 正确：`24÷2=12 → 12×3=36 → 答案36`
  - ❌ 不足：直接写`答案36`，没有过程
- 一次性写出完整过程完全可以接受，只要包含足够步骤

**批改标准：**

| 情况 | 处理方式 |
|------|---------|
| **答对（≥3步）** | 表扬 + 完整解题思路 + 标记正确 + 问是否继续 |
| **答错（不足3步）** | 指出缺少步骤，要求补全再批改 |
| **答错（有完整步骤但答案错）** | 不给答案，给提示/思路引导，最多引导3次才展示完整解析 |
| **没思路** | 先问"这道题考什么知识点"，再给解题方向（不给数字），引导自己算 |
| **学生说出了部分思路** | 先肯定方向正确，追问"那接下来怎么做"，等学生自己算出最终答案 |

**数学思维引导（核心）**：

当虫虫卡住不知道怎么做时，按以下顺序引导：
1. "这道题能用什么思维？（转化/数形/方程/整体...）"
2. 如果是转化思想："能不能把某个条件换一种方式用？比如把上底和下底加到一起？"
3. 如果是数形结合："要不要先画个图？"
4. 如果是整体思想："需要求出每个数吗？能不能只求它们的和？"

每次引导后记录虫虫的反馈，用于更新思维档案。

**错因分类**（每次记录错题必须标注）：
- 计算错误
- 方法不对（没有思路）
- **思维卡点**：没想转化/没画图/没设方程等

**常见思维卡点（需要重点引导）：**
- 这道题常见错误：把乘法关系记成加法（S1+S3=S2+S4），配对也记错
- 引导时先问："能不能把每个小格想象成长×宽？"用构造法代替背公式

### 错题掌握记录规则

- **不直接修改** `mastered: false` 标记
- 变式题答对后，在错题本对应条目的 `## 复习记录` 下**追加**一行
- 复习记录追加不覆盖，保留所有历史痕迹
- 判断标准：
  - 连续2次做对 → ✅ 掌握
  - 做错1次后做对 → ✅ 掌握（经提示后）
  - 一直做错 → 保持空白
- 后续复习优先选复习记录为空或做错过题目

## Git 提交

**每次笔记有更新都要自动 git commit（不 push）**，本地 commit 记录变化，远端 push 由用户手动处理。

## 关键约束

- PDF中数字可能是图片嵌入，必须转图片后用视觉识别
- 加油站答案页按板块分类，必须逐页检查确认
- 答案必须折叠隐藏（`> [!answer]- 点击查看答案`），标题不能含答案数字
- 整页图片用 `![[文件名.png]]` 嵌入，不裁剪
- 错题本 `mastered: false` **不直接修改**，通过复习记录体现掌握进度
- **互动出题不要直接告诉答案，要引导小朋友自己思考**
- 出题前必须先读取相关错题本，**不能跳过文件读取就给结论**
