# 奥数课程知识库管理

## Vault 结构（工作目录：`C:\Users\hp\xiaochong-math`）

```
课件资料pdf/          ← 用户放入原始PDF
附件/第N讲/           ← PDF转成的整页PNG图片
知识点/               ← 知识点笔记 + 知识地图
题库/
  课后小测/           ← 逐题文字 + 整页原图 + 折叠答案
  加油站/             ← 整页原图 + 分类折叠答案 + 掌握清单
  变式练习/           ← 针对错题的变式题 + "我的答案"填写区
错题本/               ← 按讲次记录 + .base 汇总视图
积分体系/             ← 积分账本、成就墙、连胜记录、周排名
```

## 流程一：新课件整理

用户说"第N讲上完了，帮我整理"时执行。

### Step 1：PDF转图片

使用 `scripts/pdf_to_images.py` 脚本，或手动执行：

```python
import fitz, os
out = f'附件/第{N}讲'
os.makedirs(out, exist_ok=True)
for pdf_path, prefix in pdfs:
    doc = fitz.open(pdf_path)
    for i, page in enumerate(doc):
        page.get_pixmap(dpi=150).save(f'{out}/第{N}讲_{prefix}_p{i+1}.png')
    doc.close()
```

> **命名约定**：图片文件名必须带讲次前缀（如 `第6讲_课后小测_题目_p1.png`），Obsidian 按文件名全局匹配，不区分目录。

### Step 2：读取图片识别内容

- 课后小测：读题目版图片获取完整题目和数字
- 课后小测解析版：读取答案和解析
- 加油站最后一页：答案汇总页，按"举一反三/基础过关/活学活用/勇攀高峰"分类

### Step 3：生成知识点笔记

文件：`知识点/第N讲 {主题}.md`

```yaml
---
tags: [奥数, {知识领域}]
lecture: N
topic: {主题}
---
```

内容结构：核心知识点（`> [!tip]` callout）、解题策略（Mermaid 流程图）、经典题型、易错点（`> [!warning]`）、相关链接。

### Step 4：生成课后小测 & 加油站

格式模板见 [references/answer-formats.md](references/answer-formats.md)。

加油站**必须先读答案汇总页确认板块结构**，不同讲次可能缺少某些板块，必须逐页检查确认，不能只看最后一页。

### Step 5：创建错题本

文件：`错题本/错题 第N讲 {主题}.md`，初始为空，等用户报告错题后填入。

格式模板见 [references/answer-formats.md](references/answer-formats.md)。

## 流程二：记录错题

用户说"第N讲课后小测第X、Y、Z题没做出来"时执行。

1. **更新课后小测掌握状态**：做对的题 `- [x] 已掌握`，做错的题 `- [ ] 未掌握 → [[错题链接]]`
2. **写入错题记录**：格式见 [references/answer-formats.md](references/answer-formats.md)
3. **薄弱知识点分析**：在错题文件末尾添加分析 block

## 流程三：复习出题

用户说"帮我复习"或"出变式题"时执行。文件：`题库/变式练习/变式练习 第N讲 {主题}.md`，格式模板见 [references/answer-formats.md](references/answer-formats.md)。

### 出题策略

1. **同类变式题**：方法相同，数字不同
2. **逆向题**：已知和未知互换
3. **综合题**：组合多个薄弱知识点
4. **难度递进**：基础确认 → 提高挑战

优先从加油站和拓展题中选未做过的题目，其次自编。

## 流程四：互动出题（小朋友直接对话模式）

小朋友说"出一道题"或"我要做题"时执行，适用于 OpenClaw 等对话场景。

### 出题规则

1. **直接出题**，立即出一道，不需确认
2. **随机薄弱点**：从当前讲次错题本中随机选一个薄弱点，不按固定顺序
3. **一次只出一道**，等回答后再出下一道
4. **实时保存**：每道题写入 `题库/变式练习/` 对应文件
5. **不主动给提示**，等虫虫说没头绪时再给
6. **出一道问一道**，每题后问"要继续下一题吗"
7. **几何题优先用原图**：先从 `附件/第N讲/` 找，其次文字描述，最后才用 image_generate

### 答案检查规则

**三步要求**：解答必须有完整中间过程，至少3步。一次性写出完整过程完全可以。

- **答对（≥3步）**：表扬 + 完整解题思路 + 标记正确 + 问是否继续
- **答错**：不给答案，给提示/思路引导，最多引导3次才展示完整解析
- **没思路**：先问"这道题考什么知识点"，再给解题方向（不给数字），引导自己算
- **学生说出了部分思路**：先肯定方向正确，追问"那接下来怎么做"，等学生自己算出最终答案

### 错题掌握记录规则

- **不直接修改** `mastered: false` 标记
- 变式题答对后，在错题本对应条目的 `## 复习记录` 下**追加**一行
- 复习记录追加不覆盖，保留所有历史痕迹
- 判断标准：
  - 连续2次做对 → ✅ 掌握
  - 做错1次后做对 → ✅ 掌握（经提示后）
  - 一直做错 → 保持空白
- 后续复习优先选复习记录为空或做错过题目

## Git 提交

**每次笔记有更新都要自动 git commit（不 push）**，本地 commit 记录变化，远端 push 由用户手动处理。

## 关键约束

- PDF中数字可能是图片嵌入，必须转图片后用视觉识别
- 加油站答案页按板块分类，必须逐页检查确认
- 答案必须折叠隐藏（`> [!answer]- 点击查看答案`），标题不能含答案数字
- 整页图片用 `![[文件名.png]]` 嵌入，不裁剪
- 错题本 `mastered: false` **不直接修改**，通过复习记录体现掌握进度
- **互动出题不要直接告诉答案，要引导小朋友自己思考**
- 出题前必须先读取相关错题本，**不能跳过文件读取就给结论**

## 出题质量控制

### 1. 难度对齐

出题前必须先读同讲次课后小测/加油站题目确认目标难度，变式题难度应与原题相当。

### 2. 条件验证

出题后必须验证条件是否有矛盾，特别是几何题的长度关系、平行四边形/梯形的上底下底关系。条件有矛盾时必须修正后才能出题。

### 3. 自我改进

出题后如被用户指出问题，应立即记录到 `C:\Users\hp\xiaochong-math\.learnings\LEARNINGS.md`，格式为 correction 类别，包含问题描述和改进措施。
