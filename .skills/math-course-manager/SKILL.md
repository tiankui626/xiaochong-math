# 奥数课程知识库管理

管理 Obsidian vault 中的奥数课程知识库。两类用户会触发本 skill：
- **家长**：整理课件、记录错题、补充备用题、检查题目质量
- **小朋友**：出题、做题、对答案、问思路

## Vault 路径

所有文件操作基于以下绝对路径：

```
VAULT=~/Library/Mobile Documents/iCloud~md~obsidian/Documents/奥数创新班
```

## Vault 结构

```
$VAULT/
├── 课件资料pdf/          ← 原始PDF（已gitignore，处理后保留本地）
├── 附件/第N讲/           ← PDF转成的整页PNG（文件名带讲次前缀）
├── 知识点/               ← 知识点笔记 + 知识地图
├── 题库/
│   ├── 课后小测/         ← 整页原图 + 逐题文字 + 折叠答案
│   ├── 加油站/           ← 整页原图 + 按板块分类答案 + 掌握清单
│   ├── 备用池/           ← 预生成的变式题缓存（出题时取，用完补充）
│   └── 变式练习/         ← 已出过的题 + 答题记录
└── 错题本/               ← 按讲次记录 + 错题汇总.base
```

## 流程一：新课件整理

触发："第N讲上完了"、"帮我整理"、新PDF出现在课件资料pdf/

### 1. PDF转图片

用 PyMuPDF 转换，150dpi。图片命名必须带讲次前缀（如 `第6讲_课后小测_题目_p1.png`），因为 Obsidian 按文件名全局匹配不区分目录，不加前缀会导致不同讲次引用同一张图。

**一次性转换所有PDF**：用一个 Python 脚本批量处理所有相关PDF，不要分多次执行。典型的PDF集合：
- `【课件】第N讲 xxx 创新版.pdf` — 课件（通常40-52页）
- `三春 创新 课后小测 第N讲 xxx.pdf` — 小测题目（学生版）
- `三春 创新 课后小测 第N讲 xxx(教师版).pdf` — 小测答案
- `【加油站解析】第N讲 xxx 创新版.pdf` — 加油站解析
- `三春 创新 拓展题 第N讲 xxx(教师版).pdf` — 拓展题教师版

**同时生成缩小版用于视觉识别**（关键！原图4000px宽会导致读取超时或被中断）：
在同一个脚本中，转换完150dpi原图后，立即生成缩小版到 `/tmp/lecN_small/`：
- 课件页缩到1200px宽
- 小测/加油站页缩到1000px宽

```python
from PIL import Image
# 在 PyMuPDF 转换循环中同步生成缩小版
ratio = 1200 / pix.width  # 课件用1200，其他用1000
small = Image.open(full_path).resize((target_w, int(h * ratio)), Image.LANCZOS)
small.save(f'/tmp/lec{N}_small/{fname}')
```

### 2. 识别内容

PDF中的数字经常以图片形式嵌入，文字提取会丢失。转成图片后用视觉识别读取完整内容。

**必须使用缩小版图片**（`/tmp/lecN_small/`），不要直接读原图。

**分批读取，每批不超过6张**。按以下顺序：
1. 课件分5-6批（每批约8-10页），提取知识点和模块结构
2. 小测题目页 + 教师版答案页一起读（通常5页以内，可一批完成）
3. 加油站解析页一次读完（通常6页以内）

**识别时关注**：
- 课件最后1-2页通常是笔记总结页，包含完整的知识框架，对生成知识点笔记非常有用
- 加油站的板块标题（举一反三/基础过关/活学活用/勇攀高峰）可能出现在任意页中间，不要遗漏
- 手写红色批注是答案和解题过程，蓝色批注通常是补充说明
- 加油站最后一页通常是纯答案汇总页

### 3. 生成笔记

每讲生成5个文件：

**知识点笔记** `知识点/第N讲 {主题}.md`
- 核心知识点（`> [!tip]` callout）、解题策略（Mermaid）、易错点（`> [!warning]`）
- 底部链接到课后小测、加油站、错题本

**课后小测** `题库/课后小测/小测 第N讲 {主题}.md`
- 嵌入整页题目原图 `![[第N讲_课后小测_题目_p1.png]]`
- 每题：question callout + 掌握状态 + 折叠答案（标题只写"点击查看答案"，不含答案数字）

**加油站** `题库/加油站/加油站 第N讲 {主题}.md`
- 嵌入整页解析图，答案按"举一反三/基础过关/活学活用/勇攀高峰"分类折叠
- 逐页检查板块标题，有些板块标题藏在中间页容易遗漏

**错题本** `错题本/错题 第N讲 {主题}.md`
- 初始为空模板，等用户报告错题后填入

**知识地图更新** `知识点/三春创新班知识地图.md`
- 添加新讲次到课程总览表

## 流程二：记录错题

触发："第N讲第X题没做出来"、"错了X题"

1. **更新掌握状态**：做对的 `[x]`，做错的 `[ ] 未掌握 → [[错题链接]]`
2. **写入错题记录**：来源、日期、错因（计算错误/方法不对/审题不清/知识点遗忘）、正确思路、关键一句话、关联知识点
3. **薄弱知识点分析**：归纳错题涉及的知识点，推荐加油站中的对应练习题

## 流程三：复习出题（家长模式）

触发："帮我复习"、"出变式题"、"出几道题练练"

### 出题质量自检

每道题写入笔记前必须通过6项检查。这些规则来自备用池中发现的真实问题——答案与解析不一致、数字除不尽反复凑数、条件不足无法求解等：

1. **先算后写**：完整算出答案后再写题目和解析。因为先写答案再凑解析容易导致答案和解析对不上。
2. **数字预验证**：设计题目前验证关键运算能整除。带余除法验证 `(和-商-2×余数)/(商+1)` 是正整数；勾股定理只用勾股数组（3-4-5、6-8-10等）。因为三年级小朋友不会处理无理数和无限小数。
3. **条件完备性**：已知条件数量 ≥ 未知数数量，不能有隐含假设。因为"两条斜边相等"这种隐含条件会让题目无法求解。
4. **答案回代验证**：把答案代回每个条件逐一验证。这是最有效的防错手段。
5. **无图可做性**：纯文字能否理解和做出。几何计数、阴影面积等需要图的题型不出变式题，改为推荐加油站原题。
6. **输出干净**：最终文件中不保留试错过程。因为备用池曾出现大量"重新设计""不对"等草稿残留。

### 出题保存

新出的变式题统一写入 `题库/备用池/第N讲_错题变式.md`，由家长确认后留在备用池等待使用。**不要直接写入变式练习文件**——变式练习文件只在互动出题时从备用池移入。

## 流程四：互动出题（小朋友对话模式）

触发："出一道题"、"我要做题"、"考考我"、"练习一下"

### 出题流程

1. **只从备用池取题**：从 `题库/备用池/第N讲_错题变式.md` 选一道，根据错题本薄弱知识点匹配。**不要实时现场出题**。
2. **出题后移动**：从备用池移除，追加到变式练习文件
3. **备用池不够时先补充备用池**：生成新题写入备用池（必须通过6项质量自检），由家长确认后再从中取题
4. **一次只出一道**，等回答后再出下一道

### 答案检查

引导式教学，不直接给答案，因为自己想出来的记忆更深：
- **答对**：表扬 + 标记 ✅ + 问是否继续
- **答错**：给提示引导重新思考（如"这道题可以用XX方法"），最多引导3次，3次后才展示解析
- **没思路**：先问"你觉得考什么知识点？"→ 给方向 → 引导自己算

### 记录

每次互动追加到变式练习文件：题目、小朋友答案、结果、引导过程（如有）、完整解析。

### 自动Git

互动模式下每次更新自动 `git add -A && git commit && git push`。其他流程修改后等家长确认再提交。

## 流程五：补充备用题库

触发："补充备用题"、家长主动要求、备用池某讲不足3道时提醒

### 备用池格式

文件 `题库/备用池/第N讲_错题变式.md`，每题：
```markdown
### 变式题 #X-VY
**原题关联：** 错题 #X — 简短描述
**题目：** 完整题目
**答案：** xxx
**解析：** 完整步骤
**数学思维：** 转化/数形/方程/分类/归纳
```

### 规则

- 每道错题至少2道变式题
- 所有题目必须通过6项质量自检
- 生成后由家长确认才写入备用池

## 执行注意事项

这些是从实际操作中总结的经验，避免重复踩坑：

### 图片处理
- **永远不要直接读原图**：150dpi的课件原图是4000×2250px，会导致视觉识别超时或被用户中断。必须先缩小到1200px宽再读。
- **PDF转图片和生成缩小版放在同一个脚本中**：不要分两步执行，减少等待时间。
- **附件目录要提前创建**：`mkdir -p 附件/第N讲`

### 内容识别
- **每批最多读6张图**：超过6张容易超时。课件52页需要分6批读完。
- **先读课件再读小测和加油站**：课件提供知识框架，小测和加油站提供题目和答案，顺序不能反。
- **课件主题名和PDF文件名可能不同**：如课件叫"物以类聚"但实际主题是"加乘原理初步"，以拓展题/小测的标题为准确定主题名。

### 笔记生成
- **先读已有笔记的格式再写新笔记**：确保格式一致（frontmatter tags、callout 样式、链接格式等）。
- **知识地图更新要同时更新三处**：课程总览表、知识领域分类、方法索引。
- **小测的掌握状态初始全部为 `[ ] 未掌握`**：等用户报告后再更新。
- **错题本初始为空模板**：包含注释形式的模板供后续填写。
- **加油站答案汇总页的答案要逐一核对图片中的手写答案**：最后一页的印刷答案最可靠。

### 文件命名规范
- 知识点：`第N讲 {主题}.md`
- 课后小测：`小测 第N讲 {主题}.md`
- 加油站：`加油站 第N讲 {主题}.md`
- 错题本：`错题 第N讲 {主题}.md`
- 图片前缀：`第N讲_` + 类型（课件/课后小测_题目/课后小测_教师版/加油站_解析/拓展题_教师版）+ `_pX.png`
