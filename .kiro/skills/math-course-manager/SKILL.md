# 奥数课程知识库管理

## 概述

管理 Obsidian vault 中的奥数课程知识库，包括：课件PDF处理、知识点整理、题库生成、错题记录、复习出题。

## Vault 结构

```
奥数创新班/
├── 课件资料pdf/          ← 用户放入原始PDF
├── 附件/第N讲/           ← PDF转成的整页PNG图片
├── 知识点/               ← 知识点笔记 + 知识地图
├── 题库/
│   ├── 课后小测/         ← 逐题文字 + 整页原图 + 折叠答案
│   ├── 加油站/           ← 整页原图 + 分类折叠答案 + 掌握清单
│   └── 变式练习/         ← 针对错题的变式题 + "我的答案"填写区
└── 错题本/               ← 按讲次记录 + .base 汇总视图
```

## 流程一：新课件整理

用户说"第N讲上完了，帮我整理"时执行。

### Step 1：PDF转图片

```python
import fitz, os
out = f'附件/第{N}讲'
os.makedirs(out, exist_ok=True)
# 对每个PDF：课后小测题目版、解析版、加油站、拓展题等
for pdf_path, prefix in pdfs:
    doc = fitz.open(pdf_path)
    for i, page in enumerate(doc):
        page.get_pixmap(dpi=150).save(f'{out}/第{N}讲_{prefix}_p{i+1}.png')
```

> **命名约定**：图片文件名必须带讲次前缀（如 `第6讲_课后小测_题目_p1.png`），因为 Obsidian 按文件名全局匹配，不区分目录。不加前缀会导致不同讲次引用到同一张图。

### Step 2：读取图片识别内容

- 课后小测：读题目版图片获取完整题目和数字
- 课后小测解析版：读取答案和解析
- 加油站最后一页：通常是答案汇总页，按"举一反三/基础过关/活学活用/勇攀高峰"分类

### Step 3：生成知识点笔记

文件：`知识点/第N讲 {主题}.md`

```yaml
---
tags: [奥数, {知识领域}]
lecture: N
topic: {主题}
---
```

内容结构：
- 核心知识点（用 `> [!tip]` callout 高亮公式）
- 解题策略（Mermaid 流程图）
- 经典题型
- 易错点（`> [!warning]`）
- 相关链接（链接到题库和错题本）

### Step 4：生成课后小测

文件：`题库/课后小测/小测 第N讲 {主题}.md`

格式：
```markdown
> 题目原图见下方，答案折叠隐藏。做错的题到 [[错题 第N讲 {主题}]] 中记录。

## 一、填空题
![[课后小测_题目_p1.png]]

### 第1题
> [!question] 简短标题
> 完整题目文字（含具体数字）

- [x] 已掌握  （或 - [ ] 未掌握 → [[错题链接]]）

> [!answer]- 点击查看答案
> **答案**：xxx
> 解析步骤
> **知识点**：[[第N讲 {主题}#知识点名]]
```

### Step 5：生成加油站

文件：`题库/加油站/加油站 第N讲 {主题}.md`

**必须先读答案汇总页确认板块结构**：加油站答案页按"举一反三/基础过关/活学活用/勇攀高峰"四个板块分类。不同讲次可能缺少某个板块（如第5讲无基础过关标题但实际有），必须逐页检查确认，不能只看最后一页。

检查清单：
- [ ] 读取答案汇总页，确认有哪些板块
- [ ] 逐页检查是否有遗漏的板块标题（有些板块标题在中间页）
- [ ] 每个板块的题号和答案数量与图片内容一致
- [ ] 答案按板块分类折叠，掌握情况按板块分组

格式：
```markdown
## 题目（含手写解析）
![[加油站_解析_p1.png]]
...

## 答案汇总
> [!answer]- 举一反三
> | 题号 | 答案 |
> |------|------|
> | 1 | xxx |

> [!answer]- 基础过关
...

## 掌握情况
**举一反三**
- [ ] 第1题
...
```

### Step 6：创建错题本

文件：`错题本/错题 第N讲 {主题}.md`

```yaml
---
tags: [错题, {知识领域}]
lecture: N
topic: {主题}
mastered: false
---
```

初始为空，等用户报告错题后填入。

## 流程二：记录错题

用户说"第N讲课后小测第X、Y、Z题没做出来"时执行。

### Step 1：更新课后小测掌握状态

- 做对的题：`- [x] 已掌握`
- 做错的题：`- [ ] 未掌握 → [[错题 第N讲 {主题}#错题标题|查看错题记录]]`

### Step 2：写入错题记录

每道错题包含：
```markdown
### 错题 #N — 简短描述
- **来源**：课后小测/加油站 第X题
- **日期**：YYYY-MM-DD
- **错因**：计算错误 / 方法不对 / 审题不清 / 知识点遗忘
- **题目**：完整题目
- **正确思路**：分步骤写清楚
- **关键**：一句话总结核心方法
- **关联知识点**：[[第N讲 {主题}#知识点名]]
- review_count: 0
```

### Step 3：薄弱知识点分析

在错题文件末尾添加：
```markdown
## 薄弱知识点分析
> [!warning] 需要重点巩固的知识点
> **1. {知识点名}**（第X、Y题）
> - 核心方法：...
> - 建议：重做加油站第A、B题
```

## 流程三：复习出题

用户说"帮我复习"或"出变式题"时执行。

### 出题并保存到笔记

文件：`题库/变式练习/变式练习 第N讲 {主题}.md`

```yaml
---
tags: [奥数, 变式练习, {知识领域}]
lecture: N
topic: {主题}
date: YYYY-MM-DD
source_errors:
  - "错题 第N讲 {主题}#错题 #1"
---
```

每道题格式：
```markdown
### 第X题（针对的薄弱点）
> [!question]
> 完整题目

**我的答案**：

> [!answer]- 点击查看答案
> **答案**：xxx
> 解析步骤
> **知识点**：[[第N讲 {主题}#知识点名]]

- [ ] 正确
- [ ] 错误 → 记录到 [[错题 第N讲 {主题}]]
```

用户做完后说"题目做完了"，则读取笔记中"我的答案"内容，逐题对比标准答案，报告结果并更新勾选状态。

### 出题策略

1. **同类变式题**：方法相同，数字不同
2. **逆向题**：已知和未知互换
3. **综合题**：组合多个薄弱知识点
4. **难度递进**：基础确认 → 提高挑战

### 出题来源

优先推荐加油站和拓展题中未做过的题目，其次自编变式题。

## 关键约束

- PDF中的数字可能是图片嵌入，必须转图片后用视觉识别
- 加油站答案页按"举一反三/基础过关/活学活用/勇攀高峰"分类
- 答案必须折叠隐藏（`> [!answer]- 点击查看答案`），标题不能包含答案数字
- 整页图片用 `![[文件名.png]]` 嵌入，不裁剪
- 错题本用 frontmatter 的 `mastered: false` 标记，配合 `.base` 视图筛选
